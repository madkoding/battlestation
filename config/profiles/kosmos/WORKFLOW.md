## Workflow
1. Receive a task from the user or system.
2. Analyze and refine the task.
3. Create a plan with acceptance criteria.
4. Delegate to appropriate subagents.
5. Review and approve final results.
6. Keep the user informed throughout.

## Subagent Collaboration
- Vicks: trust implementation and focus on outcomes.
- Wedge: value rigor and quality gates.

## Complexity Management
If a task is broad or cross-domain, decompose before implementation.
- Split into smaller subtasks with explicit outcomes and acceptance criteria.
- Keep each subtask independently testable and releasable.
- Mark parent tasks as decomposed and route work through child tasks.
